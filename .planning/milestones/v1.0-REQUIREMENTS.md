# Requirements Archive: Ray Tracing v1.0

**Shipped:** 2026-04-22
**Core Value:** Camera behavior must stay consistent across offline and realtime rendering so the same scene and rig produce the intended image no matter which path is used.

## v1 Requirements

### Camera Models

- [x] **CAM-01**: Developer can represent each camera in a rig as either `pinhole32` or `equi62_lut1d`.
- [x] **CAM-02**: Offline CPU rendering uses the selected camera model for ray generation instead of assuming pinhole-only behavior.
- [x] **CAM-03**: Realtime GPU rendering uses the selected camera model for ray generation instead of assuming pinhole-only behavior.
- [x] **CAM-04**: Viewer camera rigs and default viewer scene setup preserve the selected camera model for each active camera.
- [x] **CAM-05**: Both supported camera models accept `fx`, `fy`, `cx`, and `cy`, with v1 distortion coefficients defaulting to zero.

### Scene & Defaults

- [x] **DEF-01**: Builtin scene definitions and YAML-backed scene configs can declare the intended camera model for each camera.
- [x] **DEF-02**: Project defaults switch to `equi62_lut1d` for newly constructed/default camera setups.
- [x] **DEF-03**: Project-wide pinhole defaults remain available, and each camera can still explicitly choose pinhole.
- [x] **DEF-04**: A utility script can compute default `fx`, `fy`, `cx`, and `cy` from image resolution and horizontal FOV.
- [x] **DEF-05**: The default pinhole intrinsics are derived from a `90` degree horizontal FOV.
- [x] **DEF-06**: The default fisheye intrinsics are derived from a `120` degree horizontal FOV.

### Verification

- [x] **VER-01**: Automated tests verify `pinhole32` math against `docs/reference/src-cam/cam_pinhole32.h`.
- [x] **VER-02**: Automated tests verify `equi62_lut1d` math against `docs/reference/src-cam/cam_equi62_lut1d.h`.
- [x] **VER-03**: Automated tests show offline and realtime paths honor the same selected camera model and default intrinsics.
- [x] **VER-04**: Existing relevant render, scene, and viewer tests continue to pass after fisheye becomes the default.

## Follow-On Requirements

### v2 Candidates

- **CAL-01**: Render presets can set output resolution dynamically instead of relying only on hardcoded defaults.
- **CAL-02**: Each camera can load or define explicit `fx`, `fy`, `cx`, and `cy` instead of only derived defaults.
- **CAL-03**: Each camera can configure model-specific distortion coefficients for `pinhole32` or `equi62_lut1d`.
- **CAL-04**: Each camera can configure its body-relative SE3 extrinsic transform through scene or preset data.
