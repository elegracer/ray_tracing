# v2.0 Requirements: Physically Grounded Realtime USD Renderer

**Defined:** 2026-07-18
**Status:** Active
**Research:** `.planning/v2.0-REALTIME-MODERNIZATION-RESEARCH.md`

## Baseline and Reconstruction

- [x] **BASE-01:** Benchmark reports distinguish wall-clock critical path from summed parallel work and never derive invalid negative overhead.
- [x] **BASE-02:** Reproducible runs record warmup, fixed workload/seed, hardware, driver, CUDA/OptiX, avg/P50/P95/P99, GPU memory, and raw artifacts.
- [x] **DENOISE-01:** The CPU clamp stub is replaced by an OptiX temporal AOV denoiser using GPU beauty, albedo, normal, flow, depth, and previous-output history.
- [x] **DENOISE-02:** Static, motion, disocclusion, resize, camera-jump, and scene-change history behavior has automated quality checks.
- [x] **GPU-01:** Viewer reconstruction and presentation remain GPU-resident; host download is optional output/readback work, not the default display path.
- [ ] **GPU-02:** Launch parameters and workers persist across frames, and acceleration structures support measured rebuild/update/refit/instancing paths.

## OpenUSD Scene Contract

- [x] **USD-01:** `SceneIR v2` carries stable prim paths, units, up axis, hierarchy, full affine transforms, visibility/purpose, time samples, and capability diagnostics.
- [x] **USD-02:** Mesh topology, normals, tangents, UV/other primvars, interpolation, indices, orientation, subdivision metadata, and material subsets are preserved.
- [x] **USD-03:** Prototypes/instances, cameras, supported `UsdLux` lights, assets, and resolved `UsdShade` bindings compile from a composed `UsdStage`.
- [x] **USD-04:** A deterministic supported subset exports to USD and passes curated semantic round-trip fixtures.
- [x] **USD-05:** Existing YAML/builtin scenes translate through an explicit compatibility frontend into the same v2 contract.

## OpenPBR Material Contract

- [x] **PBR-01:** A versioned material block matches official OpenPBR/MaterialX input names, types, defaults, and connected-input semantics.
- [x] **PBR-02:** The core evaluator implements base, specular, anisotropy, transmission, emission, opacity, thin-walled, normal, and tangent with matched evaluate/sample/PDF behavior.
- [x] **PBR-03:** Coat, fuzz, thin-film interference, transmission dispersion, non-thin-walled subsurface random walks, and thin-walled diffuse sheets execute through the shared CPU/GPU contract with matched physical/reference gates.
- [x] **PBR-04:** Color spaces, texture sampling, radiometric units, light units, exposure, and display transfer are explicit and tested separately.
- [x] **PBR-05:** Legacy diffuse/metal/dielectric/emissive/volume materials have documented translators and compatibility images; lossy mappings are reported.

## Lighting and Sampling

- [x] **LIGHT-01:** Emissive geometry and analytic/environment lights use explicit sampling distributions, solid-angle PDFs, visibility, and MIS.
- [ ] **LIGHT-02:** CPU and GPU pass PDF normalization, furnace/energy, finite-radiance, and estimator agreement tests.
- [ ] **RESTIR-01:** ReSTIR DI improves many-light quality/performance against the unbiased baseline without invalid temporal reuse.
- [ ] **RESTIR-02:** ReSTIR GI/PT and neural caching remain experimental until DI, BSDF probabilities, temporal gates, and reference comparisons are stable.

## Acceptance

- [x] **VAL-01:** The existing compatibility suite remains green throughout migration.
- [x] **VAL-02:** The version-pinned ASWF USD WG `Vehicles` scene and official OpenPBR
  v1.1.1 MaterialX examples import through the product path and produce deterministic realtime
  and offline image outputs. The required bundle contains scene-linear EXR and display PNG
  images for three bounds-fitted poses across `pinhole32` and `equi62_lut1d`, plus one
  simultaneous mixed-model four-camera orbit render. Its manifest records exact camera
  transforms, provenance, hashes, and bounded perceptual/linear error against approved
  references; unsupported-schema rejection or a render run without the complete artifact
  bundle does not satisfy this gate.
- [ ] **VAL-03:** Every performance claim compares equal-or-better quality at identical scene, camera, resolution, and sample conditions.
- [ ] **VAL-04:** RTX 3090 remains the required baseline; newer architecture features are capability-gated.
